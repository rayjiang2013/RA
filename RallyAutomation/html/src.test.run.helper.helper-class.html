repAll(self, indexes, lst, parrent_tc, variable_value_dict, tc, steps_type, search_path, setup_calls, search_index, verdict)

source code 
Parameters:
  • indexes (list) - list of indexes, for example INDEXES_SUP - referring to constants.py
  • lst (list) - list of test case related data parsed from Rally QA_TC_PARAMS_TEXT field in each Test Case
  • parrent_tc - object of parent TestCase that calls this test case
  • tc - object of class TestCase with Rally TestCase data
  • variable_value_dict (dictionary) - dictionary of all saved variables/parameters for api level test cases or functional level test cases run within this test case
  • verdict (list) - pass or fail verdicts for all test cases under the test set
  • search_path (unicode) - path to search through the variable_value_dict
  • steps_type (integer) - test case execution steps. Please refer to definition in constants.py
  • setup_calls (list) - list of setup api or functional level calls
  • search_index (integer) - provide index to determine which value to replace if it finds more than one fields during the search of variable dictionary.
Returns:
return rep_status: return True if replace successfully else return False, lst, varbs: list of variables, missing_varbs: list of variables that are not defined in extra.json or pre-defined local variables, missing_varbs_string: concatenated string of missing varbs
